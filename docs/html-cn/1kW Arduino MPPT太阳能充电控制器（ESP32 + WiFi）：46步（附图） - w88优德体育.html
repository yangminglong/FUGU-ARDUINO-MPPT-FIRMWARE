什么是 MPPT，为什么它对太阳能电池板很重要？

最大功率点跟踪 (MPPT) 是一种与具有可变功率的能源（如太阳能电池板）一起使用的技术，以最大限度地收集能量！MPPT 太阳能充电控制器是太阳能设置的必备设备。MPPT 是智能 DC-DC 转换器。它们调节电流和电压以安全地为电池充电和为逆变器供电。除了调节之外，MPPT 还使用一种巧妙的算法来跟踪太阳能电池板的最大功率点。正确的解释是技术性的，但最简单的方法

“将 MPPT 用于太阳能，可帮助您收集太阳能电池板所能提供的最大能量”

高效同步降压设计

基于同步降压的 MPPT 是最密集和节能的设计之一。高级商业级 MPPT 使用相同的电路拓扑。事实证明，您支付的不是材料，而是公司的研发成本。建造一个很便宜，但产生一个合适的设计需要大量的工作。

第 1 步：项目规格

我的目标是构建一个 DIY MPPT，我将继续将其用于我的离网太阳能装置。它必须很好，所以我在设计过程中加强了它。这包括使用 16 位 ADC 进行精密传感器测量、使用真正快速的双核 32 位微控制器进行快速系统计算、实现更高的 PWM 分辨率以及添加用于开关优化的宽可选 PWM 频率范围。还有一堆物联网遥测选项和一个开源固件，它将与我未来的 MPPT 版本交叉兼容。

技术规格：

  • 带CC-CV的MPPT扰动算法
  • 80V、30A 输入（太阳能、风力涡轮机、PSU）
  • 50V、35A 输出（锂离子、LifePO4、铅酸等）
  • 98% 的峰值转换效率（同步降压）
  • WiFi 和蓝牙 Blynk 电话应用遥测
  • 充电器/PSU 模式（可作为可编程降压转换器运行）
  • 16 位/12 位精密 ADC 测量 (ADS1115/ADS1015)
  • 自动 ACS712-30A 电流传感器校准
  • 电池和输入断开恢复保护协议
  • LCD 菜单界面（带设置和 4 种显示布局）
  • 闪存（非易失性设置保存功能）
  • 可设置的 PWM 分辨率（16bit-8bit）
  • 可设置的 PWM 开关频率 (1.2kHz - 312kHz)

记笔记：

  • 80V 输入是指太阳能电池板的绝对 Voc。稍微超出此范围可能会损坏您的 MPPT。请注意，Voc 和 Vmp 是两个非常不同的东西。
  • 目前请将电流设置限制为 30A，该设计仅在 PSU 模式下以 48V 20A 的输出进行了台架测试。
  • 就目前而言，充电模式仅在去年夏天的高峰日照时间使用 8S LiFePO4 电池组在 22V-27V 23A-19A 连续测试。
  • 非常感谢对设计的反馈，因为我是开关电子产品的新手。我很乐意纠正本教程中的错误。
  • 无论理论额定功率如何，固件都锁定为 1kW。由于此构建中缺少并联 MOSFET 和电感器线厚度，因此执行了此安全措施。无论如何，在解锁此特定版本的 1kW 上限之前，我会在未来进行未来测试。
  • 该项目的最大输入电流感应范围为 30A。无论如何，由于 MPPT 的降压内核设计，输出将被限制在 35A 的绝对输出电流。
  • 如果我们在输入端只有一个 30A 电流传感器，如何测量更高的输出电流？由于降压拓扑的特性，假设输入电压高于输出电压，则输入电流低于输出电流。一个简单的公式用于通过使用单个电流传感器来近似输出端的电流。这将在教程中进一步解释。
  • 为什么视频上说30A输出？出于安全原因，必须降低经过 35A 测试的电感器设计的额定值。MPPT 可能输出 35A，但对于这个特定项目，固件默认设置为 30A。我知道电感的饱和电流与降压的 RMS 电流不同。由于其磁粉芯软饱和特性，假设电感器的最坏情况饱和电流是测试电流的两倍。您可以在继续撰写文章时阅读有关此内容的更多信息，因为我正在记录视频教程的第 2 部分，因此某些部分尚未更新。

项目的灵感

几个月前我买了一些太阳能电池板时，我正在寻找一些 MPPT 太阳能充电控制器，结果发现它们很贵！作为修补匠，我决定自己建造一个。我知道材料会很便宜，但设计过程需要大量工作。我搜索了不同的期刊和教程。真正脱颖而出的是 Open Green Energy 的分步设计教程！（ARDUINO MPPT SOLAR CHARGE CONTROLLER）我阅读了整个教程，令人印象深刻！他确实提到了他在构建中遇到的问题以及为什么该项目从未被认为已完成。我试图想办法解决这些问题，因此六种不同的电路板修订使我最终实现了无问题的构建。

第 3 步：我早期的原型

我的旧设计基于使用 ATmega328P-AU 微控制器 IC 的自制 Arduino Nano，类似于开放式绿色能源所使用的。自从我开始设计更高功率的 MPPT 以来，我发现 Arduino Nano 有很多限制。另一方面，它们足以降低 MPPT 设计的功耗。

我的一些原型的规格低于我在稳定版本中使用的规格，而另一些则具有更高的规格。我尝试了不同的权衡，试图在成本和性能之间找到适当的平衡。

为什么我停止使用 Arduino Nano 进行高功率 MPPT：

  • Arduino Nano 是一个 8 位微控制器，它在处理数学运算时相对较慢。请记住，MPPT 是智能且备受赞誉的 SMPS 转换器（降压转换器除外）。它需要能够在运行一堆进程时处理大量数字。这些过程包括监管、MPPT 跟踪、安全保护、遥测、计量等等。虽然我发现 Arduino Nano 的处理速度足够，但它在快速调节方面做得很差，如果输入电压上升，输出对限制电压或电流的响应相对较慢。我在我早期的原型上解决了这个问题。每当 Arduino Nano 跟不上并超出参数限制时，MPPT 充电就会停止并尝试恢复。这经常触发并扰乱我的能量收集。在下一步我'
  • Arduino Nano 和 uno 中使用的 ATmega328P 具有 10 位 ADC。这被大多数人称为模拟引脚。10 位意味着它可以表示从 0V 到 5V 的 1024 个值（5V 是默认电压参考）。当 ADC 引脚与电压和电流传感器设计用于宽电压范围（例如：80V 30A）时，您只有 1024 个值来表示 0V-80V 和 0A-30A。这为我们提供了 78mV 的电压感应分辨率和 29mA 的电流感应分辨率。这意味着您有一种非常粗略的方式来检测电压和电流。
  • Arduino UNO/ Nano 也仅限于 8 位 PWM 分辨率和 62.5kHz 的最大频率。频率还不错，但如果您想要良好的性能，8 位 PWM 分辨率是不够的。8 位 PWM 分辨率在占空比范围内为您提供 256 个唯一值。当您将其应用于高功率 MPPT 或降压设计时，值的单个变化将导致电压的大步长。这可以通过使用更大的电容器作为缓冲器并加快基于 MCU 的 SMPS 电路的循环周期处理时间来补偿。
  • 归根结底，有一种更便宜的替代品，称为 ESP32。在接下来的几个步骤中，我将解释我为什么要使用它。

双电流传感器版本：

  • 我早期的原型之一使用双电流传感器配置，一个在输出端，一个在输入端。这是必不可少的，因为降压转换器的输入电流与输出电流不同。在降压型 MPPT 中，输出端的电流通常高于输入端的电流。这意味着我需要两个不同的电流传感器来优化两个不同的范围。我尝试了 ACS712-30A、ACS758-100A、LEM DLSR50-50A 等……它们都运行良好，但它们也使构建成本更高。
  • 输入端的单个电流传感器绰绰有余。您可以通过获取输入电流和输入电压来简单地推导和近似电流输出，并将其等同于输出电压和输出电流，同时求解输出电流。（三个给定值，一个未知）。这来自假设“Power In = Power Out”。这有一个缺点。所有降压转换器都有功率损耗。由于未考虑功率损耗，因此来自输入电流传感器的输出电流将不准确。但请记住，在一天结束时，您仅使用输出电流来限制充电电流和进行库仑计数。在降压转换器中，输出功率总是低于输入功率。这意味着我将充电电流设置为 30A，您将获得的实际电流小于 30A（可以说是 29.5A 左右）。不用说，您不会在输出时受到一点电流差异的影响。

更好的电流传感器：

  • ACS712 一直是修补工使用的最流行的电流传感器之一。仅仅是因为它便宜，有效，可以双向感应电流，具有隔离性并且易于使用。但它并不完美。一方面，它很吵！即使您测量的是纯扁平直流电路（电池和负载电阻），也会有如此多的噪声，这些值可能会有点抽搐。如果您使用的是模块，您可以按照 Allegro 数据表中的说明将滤波电容更换为更大的滤波电容来限制带宽，从而降低噪声。稍后会解释该模组。另一个问题是它使用 5V Vcc 引脚作为电压参考，因此如果您的 5V 稳压电源线电压发生变化，电流传感器的模拟输出也会发生变化。这是一个很大的转折点，在设计过程中需要注意。
  • 升级！在我最近的一个原型构建中，我使用了一个名为 LEM HLSR50 的工业级电流传感器！这是我在 MPPT 上尝试过的最好的之一。它具有 ACS712 的所有优点，但没有任何缺点。它可以测量50A（峰值125A）的双向电流，它是隔离的，也很容易使用。但这一次它几乎没有噪音！模拟输出在纯直流测试中几乎没有抽动。它有一个内部电压基准，因此模拟输出独立于 5V Vcc 线路。它还有一个单独的引脚用于电压参考。这意味着可以使用差分 ADC，差分 ADC 无需计算双向电流传感器中的零点。这也意味着 HSLR50 几乎不需要传感器校准。它也被磁屏蔽了！EMI几乎不是问题，与 ACS712 不同，即使将其靠近钕磁铁也几乎不会影响性能。只有一个缺点，价格。在我居住的地方，在 e-gizmo 上只需 6 美元，但在美国或其他西方国家，从 DigiKey 购买的价格约为 16 美元。这是我没有在当前版本中使用它的主要原因。也许，有一天你会在我的未来构建中看到它。

降压转换器一般

降压转换器是将较高电压输入转换为较低电压输入的稳压器。这不是相反的方式，因为它被称为升压转换器。与线性稳压器（如臭名昭著的 7805 IC）不同，降压稳压器不会通过热量释放电压和电流以进行调节。无论是降压型、升压型还是降压-升压型，这些稳压器都尽最大努力节省功率（功率输入 - 功率损耗 = 功率输出）。请注意“负功率损耗”，尽管降压稳压器效率很高，但它永远不会达到 100% 的效率并且仍然会有损耗，除非您生活在一个乌托邦世界中。此外，降压稳压器会产生电流放大效应（有点），因为随着降压输出端的电压降低，它可以在其输出端提供的电流可能大于在输入端提供的电流。

异步降压转换器 - 示例

当“降压转换器”这个词出现时，人们首先想到的是那些便宜的蓝板LM2596降压转换器模块。大多数人都熟悉的这些模块基于效率较低的异步降压转换器拓扑。它使用一个二极管和一个 MOSFET 或 BJT（位于带有 PWM 反馈驱动器的 LM2596 芯片内部）。这些东西通常达到 75-85% 的平均效率。除了具有较低的效率等级外，您还会注意到这些模块即使在使用其宣传的额定电流时也很容易发热。这种发热是由效率低下（开关损耗、传导损耗等）引起的热耗散引起的。

异步降压转换器 - 为什么效率低下？

不要误会我的意思，与线性稳压器相比，异步降压转换器仍然有效。但它的效率可能会更好！这些异步降压器无法达到接近 98% 的效率等级的原因是由于二极管的存在而导致的损失。二极管对于降压转换器至关重要，因为它可以防止电流回流。有一种叫做正向压降 (Vf) 的东西，大多数硅基二极管的正向压降范围为 0.4V-1.2V。而已？好吧，这听起来并不多，但是当您接近大功率电子产品领域时，提到的电压降会导致大量功率损失。二极管的电压下降不仅仅是电压下降，它会产生热量，而热量在电子设备中是一种无用的能量形式。把它想象成你家的水管漏水，你不会从你的仪表测量的水量中得到相同的水量，结果你会失去水并支付更高的水费。与太阳能装置类似，浪费电力并且您无法从太阳能电池板的生产中获得最大收益。

异步降压缺乏效率的答案是对异步降压的轻微修改。通过简单地用 MOSFET 替换二极管，几乎可以消除由二极管引起的电压降。为什么是 MOSFET？与 BJT 晶体管不同，MOSFET 在激活时没有二极管的电压降。当 MOSFET 导通时，通过在栅极和源极引脚上施加电压，MOSFET 导通。您可以将 MOSFET 视为机械开关。他们也真的很快！想想 Marvel 的 Flash 在纳秒内打开和关闭电灯开关。现在我们已经解决了二极管问题引起的电压降，下一步是什么？通常的做法是使用 N 沟道 MOSFET 而不是 P 沟道 MOSFET。MOSFET 具有称为 Rds(on) 的东西，也称为导通电阻。当 MOSFET 导通时，它现在的行为就像一个电阻器，电阻器也会导致电压下降。与 P 沟道 MOSFET 相比，N 沟道 MOSFET 具有极低的导通电阻。因此，与使用二极管（大多数情况下）相比，N 沟道 MOSFET 具有显着降低的压降效应。现在，我们以热量形式消散的功率损耗减少了。操作简单，Q2导通Q3关断，Q3导通Q2关断。当然，必须实现死区时间，以防止 Q2 和 Q3 同时导通，导致快速短路（交叉传导）。简单？不，还有更多！因此，与使用二极管（大多数情况下）相比，N 沟道 MOSFET 具有显着降低的压降效应。现在，我们以热量形式消散的功率损耗减少了。操作简单，Q2导通Q3关断，Q3导通Q2关断。当然，必须实现死区时间，以防止 Q2 和 Q3 同时导通，导致快速短路（交叉传导）。简单？不，还有更多！因此，与使用二极管（大多数情况下）相比，N 沟道 MOSFET 具有显着降低的压降效应。现在，我们以热量形式消散的功率损耗减少了。操作简单，Q2导通Q3关断，Q3导通Q2关断。当然，必须实现死区时间，以防止 Q2 和 Q3 同时导通，导致快速短路（交叉传导）。简单？不，还有更多！

现在同步降压有两个主要挑战。现在我们已经用第二个 MOSFET 替换了二极管，需要解决一个新问题，主要是正确切换它们。异步降压转换器易于实现的原因是二极管无需任何花哨的电路或代码即可自动阻止电流回流。另一方面，同步降压上的 MOSFET 需要彼此同步切换，否则会烧毁，因此称为同步。

第一个问题与正确触发 N 沟道 MOSFET 有关。Q3 不是问题，因为排水管直接接地。一个简单的图腾柱驱动器就足以通过在 Q3 的栅极和源极引脚之间提供电压来切换栅极。另一方面，Q2 在顶部，其源极引脚连接到 Q3 的漏极引脚。这种配置称为半桥。要正确切换 Q2，必须使用 MOSFET 驱动器，特别是具有电荷泵的驱动器。对于不熟悉接地参考重要性的其他人来说，这可能没有意义。电荷泵使用电容器每隔一段时间充电，同时在高端 MOSFET Q2 上有一个图腾柱驱动器进行开关。这可确保 MOSFET 获得完全开启所需的电压。虽然我们知道 MOSFET 可以充当开关，MOSFET 也可以充当可变电阻器（在有源区工作）。当 MOSFET 在栅极和源极引脚上提供低于指定 Vth（阈值电压）的电压时，就会发生这种情况。电荷泵 MOSFET 驱动器确保 MOSFET 饱和并获得完全导通所需的电压，并具有尽可能低的导通电阻，从而减少降压中的功率损耗（Vgs 通常额定为 10V）。

让我们先解释一下 MOSFET 驱动器

在我找到问题的解决方案之前，我需要解释一下 MOSFET 驱动器是如何工作的，以避免让大家感到困惑。众所周知的 IR2101 和 IR2110 驱动器是我称之为原始逻辑 MOSFET 驱动器的东西。您有两个逻辑输入 HIN 和 LIN，用于控制高侧和低侧 MOSFET（Q2 和 Q3）的开关。当 HIN 为高电平时，Q2 开启，当 HIN 为低电平时，Q2 为低电平。当 LIN 为高电平时，Q3 开启，当 LIN 为低电平时，Q3 为低电平。这需要两个 PWM 互补信号用于降压操作。该芯片使您可以更好地控制开关，因为您也可以控制死区持续时间。我喜欢这款芯片，因为我可以直接独立控制 Q2 和 Q3。

最容易使用的 MOSFET 驱动器是 IR2104 IC，它也使用了开放式绿色能源。现在让我们更多地关注这一点，因为它是我在最新版本中使用的。IR2104 具有内置的死区时间功能，使编码过程更加容易。IR2104 具有逻辑输入引脚名称 IN 和 SD。IN 是用于开关的 PWM 信号的输入逻辑，SD 是使能引脚。SD只是一个覆盖一切的引脚。当 SD 为低时；无论 IN 是否存在 PWM 信号，Q2 和 Q3 都会关闭。当 SD 为高电平时，Q2 或 Q3 将打开，具体取决于 IN 引脚逻辑状态。当 IN 为高时；Q2 为高，Q3 为低。当 IN 为低时；Q2 低，Q3 高。这是什么意思？好吧，现在我们只需要 1 个 PWM 信号来反向切换 Q2 和 Q3，从而减少对两个互补 PWM 信号的需求。这意味着 PWM 占空比 % 越高；Q2 开启时间越长，Q3 开启时间越短。另一方面，PWM占空比%越低；Q3 开启时间越长，Q2 开启时间越短。在这一点上，你们中的一些人可能对问题可能是什么有了更清晰的了解。

解决方案：

这个问题有两种解决方案（硬件与软件），要么为配备回流电流控制的同步降压转换器实施特定应用的 MOSFET 驱动器 IC（Analog Devices 提供大量此类解决方案），要么简单地使用旧的 IR2104 设计和修复它通过代码。显然，我选择了 IR2104 设计并用代码对其进行了修复，因为 IR2104 非常便宜，而且在任何地方都不是同步降压驱动器专用芯片的 10 美元价格标签。

我发现 PWM 占空比 % 有一个特定的底值，当电池连接到输出端时，它不应该是 0% 或低于指定的 PWM 底值。那么我指的 PWM 底值是多少？我想出了一个简单的公式，我也实现了我编写的开源固件。

  • PWM 下限占空比 = (输出电压 / 输入电压) * 100

这意味着最低可能的 PWM 占空比不应低于“PWM 下限占空比”。该公式采用输出电压和输入电压比来计算降压所需的理想空载 PWM 占空比，以输出与输出端连接的电池电压相匹配的电压。当 PWM 信号的占空比低于计算出的 PWM 最低值时，电流会反向流动并导致 Q3 在不应该导通时导通。因此，您实际上不是在给电池充电，而是在耗尽它。当您从计算出的 PWM 下限偏离到较低的 PWM 信号时，反向电流 Q3 会变得越来越高，从而导致其烧毁。在代码中，当 PWM 低于计算的下限时，IR2104 的 SD 引脚变为低电平，以确保 Q2 和 Q3 在此状态下永远不会开启。您会注意到“PWM Floor Duty Cycle”被命名为 PPWM 并添加了“PPWM_margin*pwmMax”，这是为了限制最大允许的 PWM 占空比，因为像 IR2104 这样的电荷泵 MOSFET 驱动器不能以 100% 的占空比运行，因为其IC设计。它还将占空比 % 转换为 anlogWrite 或 ledcWrite 函数接受的整数值。

  • PPWM =(PPWM_margin*pwmMax*voltageOutput)/(100.00*voltageInput);
  • PPWM = 约束（PPWM,0,pwmMaxLimited）；
  • PWM = 约束（PWM，PPWM，pwmMaxLimited）；
  • ledcWrite(pwmChannel,PWM);
  • buck_Enable();


Arduino 中的约束功能限制 PWM 信号超出上限以及低于下限。函数“ledcWrite(pwmChannel,PWM)”是 ESP32 等效于 Arduino Uno/Nano 的模拟写入，该函数用于设置 GPIOx 引脚上的 PWM 信号。上面两行之间有很多代码，我只是写了最重要的代码来突出修复解决方案。

第 9 步：为什么我使用外部精密 ADC (ADS1115 - Texas Instruments)

Arduino Nano 中的 ADC

Arduino Nano 和 uno 中使用的 ATmega328P 具有 10 位 ADC。这被大多数人称为模拟引脚。10 位意味着它可以表示从 0V 到 5V 的 1024 个值（5V 是默认电压参考）。当 ADC 引脚与电压和电流传感器设计用于宽电压范围（例如：80V 30A）时，您只有 1024 个值来表示 0V-80V 和 0A-30A。这为我们提供了 78mV 的电压感应分辨率和 29mA 的电流感应分辨率。这意味着您有一种非常粗略的方式来检测电压和电流，并且您的电压和电流可能会以 78mV 和 29mA 的间隔跳跃。

ESP32 中的 ADC

ESP32 有一个内置的 12 位 ADC，这为您提供了 4096 个值来代表您的电压和电流。这比 Arduino Uno/Nano 高四倍！对于 80V 30A 设置，我们得到 19.5mV 的电压感应分辨率和 7.32mA 的电流感应分辨率。这是来自 Arduino Uno 和 Nano 的更好的电压和电流感应分辨率！ESP32 也有更快的处理器！在监管方面，它也将能够更快地进行轮询，使其更具响应性。但是有一个缺点！ESP32 以有一个糟糕的 ADC 着称！它具有非线性 ADC 响应。这意味着 ADC 不能准确地表示 0V 和 Vref 之间的电压。就足够了，但我不想就此解决。解决方案？

外部精密 ADC

解决方案是使用外部精密 I2C ADC！我选择的是 Texas Instruments 的 ADS1115 或 ADS1015。为什么要使用外部 ADC？这些外部 ADC 是为特定目的而设计的，即仅用作 ADC，仅此而已，而且它的工作非常出色！注意当您将传感器值连接到 Arduino Uno 时，它们是如何意外抽动的？尝试使用外部 ADC，您会注意到值几乎不会意外抽动。

ADS1x15 ADC 具有内置的内部电压基准。这意味着您的模拟值读数独立于 ADC 的 Vcc。这使其不易受系统电源线噪声的影响。它具有内置的可编程运算放大器，这意味着您可以选择各种增益。最重要的是，ADS1115 具有 16 位 ADC 分辨率！16 位 860sps！16 位会给你一个 65,5536 来代表你的电压和电流！这比 Arduino Uno 的 ADC 好 64 倍。对于 80V 30A MPPT 设置，我们现在具有 1.22mV 的电压感应分辨率和 0.457mA 的电流感应分辨率！太疯狂了！

我应该选择 ADS1115 还是 ADS1015？

最大的问题是，我需要哪一个？尽管规格略有不同，但这两款芯片都与 Adafruit 的 Arduino ADS1x15 库交叉兼容。我得到的是 16 位分辨率和 860sps 速度的 ADS1115 还是 12 位分辨率和 3,300sps 速度的 ADS1015？正如视频中提到的，我已经尝试了它们。这个版本不需要 ADS1115，它的分辨率对于 80V 30A 来说是我几乎不需要的。我使用 ADS1015 进行整个项目的测试和文档。

为什么选择 12 位 ADS1015？

ADS1015 具有 12 位 ADC 分辨率，与 ESP32 的 12 位 ADC 分辨率相同，为什么还要选择 ADS1015？答案是稳定性。记得当我对董事会进行不同的修订时。我试过 Arduino Nano、Arduino Uno 和 ESP32；他们所有的 ADC 都不如 ADS1x15 等专用外部 ADC 稳定。即使解锁 ATmega328P 和 ESP32 的内部 ADC Vref（用于稳定性的模块），与外部 ADC 相比也算不上什么。请记住，现在我们正在使用更高的 PWM 分辨率，我们现在需要感应电压和电流的微小变化，以使 MPPT 扰动算法完美运行。传感器读数中的轻微噪音和抽搐会严重影响 MPP 跟踪。外部ADC是要走的路！

MOSFET：

  • 该项目最终具有极小的外形尺寸和高能量转换效率的关键原因之一是仔细选择了 N 沟道 MOSFET。CSD19505 是我能找到的最好的低成本 MOSFET 之一！导通电阻为 2.6mΩ，Vds 为 80V，这简直就是野兽！这意味着更少的传导损耗。
  • 单个CSD19505相当于三个IRF3205并联，可以处理更高的80V电压。如果我们要使用类似 IRF3205 的东西，我们最终会使用其中的九个来代替三个 CSD19505。CSD19505 的栅极电荷也显着降低，这也有助于降低开关损耗。
  • 在寻找 MOSFET 替代品时，请找到 Vgs 为 80V 或更高的 MOSFET。Id 必须高于 90A。Rds(on) 必须尽可能低（在不为此 MPPT 使用并行配置的情况下选择单个 MOSFET 时，我不会高于 4mΩ）。较低的 Qg 总是更好。

微控制器：

  • U8 是 ESP32 代表系统的微控制器
  • R25-R28 是按钮的下拉电阻，以防止引脚悬空
  • R29 & R39 是 I2C 端口的上拉电阻
  • C19 是 ESP32 编程中臭名昭著的自动编程问题的电容器修复
  • C23 是一个简单的旁路电容器，用于降低 ESP32 的 3.3V 线路上的纹波
  • R21 & R24 是 ESP32 的 EN 引脚的上拉电阻（对于 UART 也是必不可少的）
  • LED1 和 R36 是一个简单的 LED 指示灯，连接到 ESP32 的默认 LED 指示灯引脚

USB-TTL UART：

  • U9 是一个 CH340C USB-TTL UART 芯片，用于 USB 串行通信和通过 USB 编程 ESP32
  • U4 是一个 3.3V 稳压器，用于 USB 端口的 5V 输入。这是必需的，因为我们的 ESP32 使用 3.3V 逻辑
  • C21 和 C22 是 U4 稳压器的标准大容量电阻器
  • D7 是一个具有低 Vf 的肖特基二极管，用于在系统由太阳能电池板或电池供电时防止电流回流到 USB 端口。1N4007 可用作替代品。

外部ADC：

  • U10 是一个外部 ADS1015 12 位 ADC（前面的步骤提到了有关此的详细信息）
  • C20 是一个标准旁路电容，用于降低 U10 的线路噪声
  • R31 是 U10 的警报引脚的上拉电阻。警报引脚是一个可编程比较器，我计划在未来的固件更新中使用它，截至目前它尚未使用。
  • A0-A3 是外部 ADC U10 的模拟输入。
  • U10 有一个等于 2.048V 的设定电压参考 (Vref)，这被普遍设置为适合 U1 的模拟输出范围（ACS712-30A 电流传感器）。
  • U10 是一款高度稳定、精确和准确的外部 ADC，与使用 MCU 的内部 ADC 相比，它可以提供更清晰的传感器值。

电压传感器：

  • 使用分压器可以轻松实现电压检测。
  • R1 和 R2 构成一个分压器，输入范围为 0-80V，分压器输出范围为 0-1.989V。这是一个低于外部 ADC U10 的 2.048V 参考电压的电压。它接近 Vref 以最大限度地提高 ADC 分辨率，但不太接近可能导致削波。
  • C1 是一个旁路电容，用于过滤输入分压器的输出噪声。
  • 对于 0-80V 输出范围，R32、R33 和 R34 最初与 R1 和 R1 具有相同的值。是的，MPPT 最初可以将电池化学物质充电至 80V 而不是 50V。然而，我将其降低到 50V 以提高输出分压器的 ADC 分辨率。人们一开始几乎不使用高于 48V 的电池。
  • R32、R33 和 R34 构成一个分压器，输入范围为 0-50V，分压器输出范围为 0-2.04V。这是一个低于外部 ADC U10 的 2.048V 参考电压的电压。它接近 Vref 以最大限度地提高 ADC 分辨率，但不太接近可能导致削波。
  • C13 是一个旁路电容，用于过滤输出分压器的输出噪声。

电流传感器：

  • U1 是 ACS712-30A 双向、隔离式 30A 额定电流传感器 IC。
  • U1 是双向的，但我们只对将其用作单向电流传感器感兴趣，以最大限度地提高电流检测的 ADC 分辨率。U1 的-IP 和+IP 引脚反向连接的原因是它的负电流响应。没有电流时 Vout = Vcc/2。这意味着当没有电流通过电流感应引脚 -IP 和 +IP 时，电流传感器的模拟输出为 5V Vcc 的一半，约为 2.5V。当电流从 -IP 流向 +IP 时；Vout = 2.5 - (电流检测 * 0.066)。我们现在知道在 -30A 时，Vout = 2.5V；在 0A 时，Vout = 0.52V。现在，我们有一个接近地面的地板和一个 2.5V 的上限，我们可以按比例缩小到 ADC 的参考电压。这样我们就不会
  • R3 和 R4 是一个分压器，可将我们从 U1 的模拟输出获得的 2.5V 上限按比例缩小到略低于 U10 的 2.048V 参考电压的电压。这可以防止传感器读数被截断。
  • C2 是 U1 (ACS712) 数据表中提到的滤波电容器。增加 C2 会导致 U1 的电流感应带宽减少。由于我们处理的是直流，我们只需要低带宽来抑制可能干扰我们电流感应的噪声。建议 10uF 用于 DC 应用的最大滤波，但数据表中也提到 10uF 会产生较慢的上升和响应时间。我最终选择了 2.2uF。
  • R5 和 C9 是一个 RC 滤波器，可进一步降低 U1 的模拟输出拾取的噪声。
  • 请注意我是如何特别注意电流传感器的。从我过去的 MPPT 原型中，使用 ACS712 模块，库存值被认为比这个修改版本更嘈杂。实施此简单修复后，MPP 跟踪性能显着提高。

温度感应器：

  • R35 是一个 10k NTC 热敏电阻，用作温度传感器
  • R12 是一个下拉电阻，它与 R35 形成一个分压器，以创建一个简单的温度传感器
  • C12 是一个旁路电容，用于对温度传感器的模拟输出进行滤波。
  • 温度传感器单元的模拟输出连接到 GPIO35（使用 ESP32 的内部 ADC）。我没有将温度传感器连接到外部 I2C ADC，因为它不需要太多精度。ESP32 也有一个更快的 ADC 用于这个不需要太多优先级的传感器。

回流电流控制单元 (BCCU)：

  • 在前面的步骤中详细提到了回流电流控制单元
  • Q1 是一个反向阻断、高侧 N 沟道 MOSFET，它提供了 Q2 的体二极管电流泄漏的可控阻断效应。
  • R37 是一个下拉电阻，用于在 BCCU 关闭时释放 Q1 的栅极电荷。
  • B1212 是一款隔离式 12V DC-DC 转换器。这用于为切换 Q1 的源极和栅极引脚提供单独的接地电位。
  • 当 GPIO27 为高电平时，Q4 导通，U2 从 12V 线路获得电源。然后向 Q1 提供一个隔离的 12V 电源以开启和导通。
  • 当 GPIO27 为低电平时，U2 和 Q1 的电源被切断，R37 释放 Q1 栅极和源极引脚上的剩余电荷，导致其关闭。
  • C4 和 C3 是数据表推荐的 U2 旁路电容器。
  • R6 和 LED3 是一个简单的 LED 指示灯，用于指示 BCCU 何时处于活动状态。

线路降压稳压器：

  • U5 和 U6 是 80V 0.4A 降压稳压器，用于为 MPPT 系统中的所有其他组件提供稳定电压。
  • U6 的固定输出由 R17、R18 和 R19 设置为 3.3V。这提供了所有 3.3V 组件
  • U5 具有由 R14、R15 和 R16 设置为 10.625V 的固定输出。这为冷却风扇端口、BCCU 和 MOSFET 的驱动器栅极驱动电源引脚提供 10.625V。最初设置为 12V，但我不得不降低它以减少开关损耗并减少冷却风扇的消耗。尽管如此，所有组件仍可使用 10.625V
  • U3 是一个线性稳压器，连接在 U5 的 10.625V 输出端。这提供了 U1 运行所需的 5V。我知道由此产生的损失，但可以忽略不计，因为 U1 需要的电流非常小。U5 和 U1 的较大电压差也可能是一件好事，因为它能够更好地调节 5V 线路，因为 U1 的模拟输出稳定性在很大程度上取决于 5V 线路调节。

MPPT 主同步降压：

  • 这部分也是从前面的步骤中详细解释的（这里还是总结一下）
  • Q2 和 Q3 是高侧和低侧 N 沟道 MOSFET，形成用于开关的半桥。
  • L1 是 MPPT 同步降压的电感器，这是一种非常快速的能量存储设备。
  • C7 和 C8 是大容量或旁路电容器，用于从 SMPS 降压单元的快速开关特性引起的纹波电压中过滤输入和输出。
  • U7 是配备电荷泵的 IR2104 半桥 MOSFET 驱动器
  • R8 & R11 为下拉电阻，用于防止 Q2 在启动前悬空
  • R9 和 R10 是栅极电阻器，用于限制 U7 为触发 Q2 和 Q3 的栅极引脚而提供的瞬态电流。
  • D1 和 D2 是肖特基二极管，它们提供了一条快速返回路径，用于在 Q2 和 Q3 中的任何一个关闭时排出它们的栅极电荷。
  • D3 是一个肖特基二极管，以确保电流永远不会流回为 C10 充电的 12V 电源线
  • C10 是 U7 的电荷泵使用的自举电容器，用于为高端 N 沟道 MOSFET Q2 提供适当的电源。
  • C11 是数据表推荐的 U7 旁路电容
  • R13 和 R20 是必不可少的下拉电阻，可防止 U7 的逻辑引脚悬空。这是对 DIY MPPT 设计中长期存在的问题的又一重要修复。如果没有这个，当您的 MPPT 运行时，将 USB 电缆连接到您的计算机会导致 MCU 的引脚在运行期间浮动。因此，MPPT 的降压部分有时会出现故障，并意外打开 Q2 或 Q3，从而产生一些神奇的烟雾。一定要添加这些电阻，这样可以确保 MPPT 的降压在 GPIO 引脚浮动时关闭。

第 16 步：电路板组装（烙铁法）

如视频所示，您可以使用多种技术来焊接 SMD 组件，而无需使用焊膏和热风回流焊枪。这就是我在 2017 年进入 SMD 时焊接 SMD 组件的方式。

一般技术：

  1. 焊接单个焊盘
  2. 用镊子夹住你的组件
  3. 将其与焊盘对齐
  4. 熔化您之前焊接的镀锡焊盘。这充当了组件的锚
  5. 将组件的所有剩余脚焊接到焊盘上
  6. 如果表面上的助焊剂有助于焊接，一点点
  7. 如果助焊剂和焊料产生烧焦或烧焦的残留物，请使用牙刷并用酒精清洁电路板

第 17 步：电路板组装（热风法）

我最喜欢的下一个技术是手持式回流 SMD 焊接技术。我的第一个是什么？如果是我的第一个 PID 烤箱，但我今天不会展示它。让我们使用最简单的 SMD 焊接技术。

手持焊接回流技术：

  1. 拿起焊膏注射器
  2. 在 PCB 的焊盘上注入一团焊膏
  3. 将您的组件与焊盘对齐
  4. 用热风回流焊枪加热焊盘

  7. 而已！简单！

第 20 步：正确安装散热器（需要隔离）

如果您是 TO-220 封装晶体管的新手，请阅读本文并特别注意。使用 TO-220 封装的 MOSFET 始终将其“漏极”引脚连接到 MOSFET 接线片，是的，它们是电连接的。直接拧紧 MOSFET 会导致散热器与 MOSFET 的所有漏极引脚一起传导。

现在让我们回顾一下原理图。MOSFET Q1 和 Q2 具有首先连接的漏极引脚，两个 MOSFET 之间不需要隔离。另一方面，MOSFET Q3 有一个“漏极”引脚，不应与 Q1 和 Q2 的漏极引脚导通。Q3 是我们需要与散热器隔离的 MOSFET。

散热器组装步骤：

  1. 首先将 MOSFET 焊接到电路板上。
  2. 抓住你的散热片并与你的 MOSFET 放在一起
  3. 使用记号笔标记要钻孔的孔
  4. 我使用的是 3.2 毫米螺栓，所以使用比螺栓尺寸略小的钻头
  5. 钻穿铝制散热器。
  6. 在进行隔离步骤之前，用力将螺栓拧到散热器上，以在散热器上形成螺纹。这是一种简单、野蛮且有效的方法，无需使用攻丝工具即可在散热器上设置螺纹或凹槽。

隔离步骤：

  1. 将塑料绝缘套管插入 MOSFET 的孔中，或者将其套在螺栓上。
  2. 在 MOSFET 和散热器之间添加玻璃纤维或云母隔离垫。
  3. 用 MOSFET 和散热器之间的塑料绝缘衬套拧紧螺栓

如果 Q3 是唯一需要与 Q1 和 Q2 隔离的 MOSFET，您为什么要在视频中将它们全部隔离？

我将它们全部隔离，因为我不希望散热器在任何地方连接到电路。这对我来说很重要，尤其是在董事会正在开发和接受测试的时候。如果我将 Q1 和 Q2 连接到散热器，散热器可能会从面板传导 80V 电压，这是我在为实验室设备测试探测不同引脚时不想承担的风险。杂散电线的轻微接触可能会损坏我的构造，需要更换很多零件。另一方面，仅隔离 Q3 而让 Q1 和 Q2 通过散热器传导有一点好处，这在 Q1 和 Q2 之间提供了较低的电阻路径，尽管它可以忽略不计。我建议将它们彼此隔离。

第 21 步：NTC 热敏电阻温度传感器

第 22 步：根据您的需要选择熔断器额定值

您会注意到我使用了 20A 绿色迷你汽车保险丝，而不是我在原理图中推荐的 35A。我这样做的原因是为了保护我的电路不受太阳能装置的影响。虽然 MPPT 运行大量安全保护协议，但这些协议和安全功能取决于微控制器和代码。如果微控制器发生故障并进入锁定状态（根据我的现场测试，这极不可能），保险丝可以节省大量硬件免于油炸。在我记录这个项目时，我没有合适的可焊接保险丝连接器，所以我只是将它们直接焊接到板上。

如何选择保险丝：

- F1 或 Fuse1 需要根据您的输入电流（太阳能输入）进行额定。您的 fuse1 额定值必须接近太阳能装置的组合 Imp（最大点电流）。

- F2 或 Fuse2 需要根据您的输出电流（电池充电电流）进行额定。您的 fuse2 额定值必须接近电池充电电流。（我的是 30A，但我最终将其限制为 20A，以延长电池组的使用寿命）

- 您仍然可以使用 35A 保险丝并像对待任何商业 MPPT 一样对待本装置。这是通过假设所有保险丝额定值都是为最大广告工作条件而设计的。请注意，使用远离您当前设置的电流的保险丝可能会阻止保险丝在那里发挥作用，从而使您的 MPPT 免受意外故障的影响。

电感器额定电流

您经常在预制电感器中找到的电感器电流额定值是指称为电感器饱和电流的东西。这是指在没有直流电流的情况下，电感值下降到低于其测量值的指定量时施加的直流电流。一些制造商会根据电感下降 30% 来评估他们的零件。请记住，电感器是非常快速的能量存储设备。能量通过流经电感器的电流产生的磁场存储。材料只能存储电感器可以吸收的特定量的能量。当电感器不再吸收能量时，电感器也会随着电感的减小而饱和。构建一个电感器并获得所需的匝数来获得所需的电感很容易！但是从你的环形电感设计中确定电感电流是困扰我好几个月的事情！这也不是您可以轻易遇到的东西！确定电感电流的方法有两种；一种是使用数据表和公式，另一种是使用连接到自制设备的示波器进行电感脉冲测试（稍后我将演示）。

第 27 步：测试电感器的电感（可选）

验证新建电感器的电感是一种很好的做法。从您的初始设计中得到错误的电感是可能的，尤其是当你们中的一些人依赖通用环形磁芯时。我对我的产品很有信心，因为我将它建立在一个带有数据表的品牌环形磁芯上。尽管如此，我还是进行了电感测试。我使用我的 Agilent 4263B 实验室级 LCR 仪表来验证我的。将我的电感器连接到 LCR 表的开尔文探头后，LCR 表显示 63.790uH！这真的很接近我的 64uH 目标，太完美了！

第 28 步：测试电感的饱和电流（可选）

现在我们可以验证电感的饱和电流。如上一步所述，电感器的饱和电流是决定我们的 MPPT 可以处理多少电流的因素之一。所以我需要确定我新建的电感器是否能真正处理我计算出的 33A。所以我建立了这个与示波器一起工作的电感器测试台。我们有一个用于存储能量的大型低 ESR 电容器组、一个用于切换电路的 MOSFET、一个用于电流感应的 0.1 欧姆分流电阻器、一个用于发送脉冲以向 MOSFET 发送信号的 Arduino ESP32 MCU 以及一个用于设置 MCU 脉冲持续时间的旋转编码器。

我搭建的电感器测试台的周期频率是20Hz，为了让电容器组充电，它被设计得很慢。脉冲持续时间的开启状态可设置为 1 纳秒至 30 毫秒。通过在我的示波器上添加一个 BNC 连接器并将 BNC 连接器连接到分流电阻器上，我们可以测量分流电阻器上的电压以测量电流。您在示波器显示屏上看到的曲线是流过电感的电流。

正如预期的那样，电感器具有软饱和电流。这对于粉芯电感材料很常见。饱和电流点可以在电流结束呈线性行为的点找到（在直线的末端找到）。很难说，但从零点到饱和点大约是 1.8 格。通过使用欧姆定律，在分流电阻上测得的电压和分流电阻的阻值，我们可以得到 Isat 或饱和电流。

  • Isat = Vshunt / Rshunt
  • Isat = (1.8divs * 2V/div) / 0.1Ω
  • 伊萨特 = 36A
  • 我们有一个 36A 的电感器！

事实证明，我们有一个 36A 额定电感，比我们预测的 33A 电感设计高 3A。我们的MPPT毕竟可以处理36A！

Hi, that is only a warning, because the lib is maybe not fully tested on the ESP32 architecture. If the rest is compiling fine, you can ignore it. If you don't like the warning, you can go to your library folder, open the LiquidCrystal_I2C folder and search for the file "library.properties". Open it in a text editor and search for the line "architectures=avr". Replace it with "architectures=*", that means, the compiler now assumes that lib is for all possible architectures (avr, esp32,....). Now the warning should be gone, but be aware: If you you use the lib for any other architecture that it's not intended to use, then other warnings or errors might appear because it might try to access pins or internal hardware that is maybe not existant in that chip.
